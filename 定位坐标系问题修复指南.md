# 定位坐标系问题修复指南

## 问题诊断

### 症状
1. ✅ 机器人已放置在建图时的起始位置
2. ❌ 但定位失败,fitness score频繁超过2.0
3. ❌ RViz中机器人显示在地图下方或异常位置

### 根本原因

**坐标系方向不匹配**:
- Fast-LIO建图时使用的`camera_init`坐标系 **不是** 标准ROS坐标系
- 从截图看: X朝左, Y朝下, Z朝后 (而非标准的前-左-上)
- PCD地图保存的是这个非标准坐标系下的点云
- NDT定位节点读取PCD地图后,与实时点云配准
- 但NDT假设点云在标准ROS坐标系下
- 导致配准结果错误,机器人"跑到地图下方"

## 关键检查点

### 1. 验证TF树完整性
```bash
ros2 run tf2_ros tf2_tree
```

应该看到:
```
map -> odom -> camera_init -> body -> base_footprint -> base_link
```

如果链断裂,说明某个static_transform_publisher没启动。

### 2. 检查NDT输出位姿
```bash
ros2 topic echo /lidar_localization/pose
```

观察:
- `position.z`: 应该接近0 (地面高度)
- 如果是很大的负数,说明坐标系旋转错误

### 3. 检查点云frame_id
```bash
ros2 topic echo /fast_lio/cloud_registered --once | head -10
```

应该看到:
```
header:
  frame_id: camera_init
```

### 4. 查看调试日志
在NDT节点的输出中,观察:
```
fitness score: 0.XXX  (应该 < 2.0)
delta_angle: XX.XXX[deg]  (应该稳定,不应剧烈变化)
```

## 临时调试技巧

如果定位一直失败,可以尝试:

1. **增大score_threshold**:
   ```yaml
   score_threshold: 5.0  # 临时放宽标准
   ```

2. **减小ndt_resolution**:
   ```yaml
   ndt_resolution: 0.5  # 提高精度
   ```

3. **查看NDT匹配可视化**:
   在RViz中添加:
   - `/lidar_localization/map` (PCD地图)
   - `/lidar_localization/aligned_cloud` (对齐后的点云)
   
   观察点云是否与地图对齐
